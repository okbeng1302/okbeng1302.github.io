动态网页的使用场景

  • 单页模式
    单页模式指不需要外部跳转的网页，例如个人主页。
  • 页面交互多的场景
    一部分网页上，有很多的用户交互接口，例如去哪儿的机票选择网页，用户可以反复修改查询的参数
  • 内容及模块丰富的网页
    有些网页内容很丰富，一次加载完对服务器压力很大，而且这种方式延时也会很差；用户往往也不会查看所有内容

动态网页带来的挑战

对于爬虫：

  • 简单HTML已经不行了，必须得有web容器来运行HTML的脚本
  • 增加了爬去的时间
  • 增加了计算机的CPU、内存的资源消耗
  • 增加了爬去的不确定性

对于网站：

  • 为了配合搜索引擎的爬去，与搜索相关的信息会采用静态方式
  • 与搜索无关的信息，例如商品的价格、评论，仍然会使用动态加载

分析动态网页-对比

  • 使用BeyondCompare 或者 SVN 等工具，来对比网页，大致找出加载的部分
  • 针对要提取的部分，分别查看 html only 与 full webpage，找出动态数据的部分
  • 记录下它们的class 或者 id，试着用一下代码来提取，如果不能提取，说明是动态的

\ and \

网页中，href后面的连接有三种：

  • href=”http://www.baidu.com",http:// 是完整URL，直接跳转（经常外链是绝对路径）
  • href=”\detail.taobao.com\isdkjs” // 是协议相关的绝对路径，如果出现是 https://xxx 则需要在 //detail.taobao.com 前面加上https: 前面的协议也可以是file:// 或者ftp://
  • href=”\i\837220.html” / 是网站的相对路径，需要在前面指明当前url的协议以及domain
